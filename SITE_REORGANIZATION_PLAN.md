# 可积系统研究雷达调整计划

**状态：** 主要编辑政策和页面方向已确定；等待 30 天样本及回溯名单确认后实施  
**最后更新：** 2026-07-19

相关筛选结果见 [`W28_W29_RETROSPECTIVE_SCREENING_REPORT.md`](W28_W29_RETROSPECTIVE_SCREENING_REPORT.md)。

## 1. 项目边界

本站要做的是：

> 面向熟悉可积 PDE 和谱方法的研究者，提供一个轻量、精选、可解释的近期论文雷达。

本站的核心价值是帮助读者减少需要亲自筛选的论文数量，并清楚说明：

1. 论文研究了什么问题、得到了什么主要结果；
2. 可积结构在论文中起什么作用；
3. 论文的创新是什么。

本站不建设：

- arXiv 镜像或完整论文数据库；
- 整个可积系统领域的百科全书；
- 全历史语义检索系统；
- 作者或课题组优先推荐系统；
- 自动生成并公开发布的旧文阅读链；
- 为保证每日篇数而补入的旧论文。

今后的功能只有在直接帮助“筛选近期论文”或“解释论文的可积结构与创新”时才考虑。

## 2. 已确定的编辑政策

### 2.1 不保留个人方向优先级

DNLS、Gerdjikov--Ivanov、Fokas--Lenells、coupled NLS、特定作者和课题组都不获得额外权重。

所有候选论文使用同一套标准。方程、作者、合作关系和编辑个人兴趣不能成为入选理由。

### 2.2 目标读者

主要读者是熟悉可积 PDE、Lax pair、谱方法、反散射和基本非线性波理论的研究者。

页面不需要从零解释基础概念。对量子可积性、可积概率等相邻方向，必须说明它们与经典谱、散射、层级、RHP、Painlevé 或守恒结构的桥梁。

### 2.3 入选标准

公开论文原则上同时满足：

1. 可积性是核心结构或核心方法，而不是附带背景；
2. 有清楚的新贡献；
3. 能从 arXiv、论文正文或出版商页面准确核验；
4. 能向目标读者说明结构联系；
5. 提供超过直接浏览 arXiv 标题和摘要的编辑价值。

强候选通常同时具有：

- **结构创新**：新的或实质扩展的谱问题、Lax pair、层级、Hamiltonian 结构、RHP、tau 结构、谱曲线、Painlevé 或 Yang--Baxter 结构等；
- **明确创新**：新定理、新方法、新渐近、新分类、新实验、新反演方法、新控制机制或新的宏观动力学结果。

通常不收录：

- 标准变换的常规应用；
- 主要由少量低阶显式解和参数图构成的论文；
- 没有新结构结论的数值展示；
- 熟悉方法几乎无概念变化地迁移；
- 可积性只在引言或背景出现；
- 无法可靠说明研究问题、结构作用和创新的论文。

不公开使用“较弱”“不值得读”“创新性差”等评价。未收录只表示它不符合本站本轮范围或当前说明门槛。

### 2.4 新旧由论文事件决定

公开排序依据：

- 首次公开日期；
- 重大修订日期；
- 首次正式在线期刊发表日期。

本站收录日期只属于编辑历史，不参与首页排序。

### 2.5 首页暂定最近 30 天

先完成 2026-06-20 至 2026-07-19 的完整样本，再判断最近 30 天的实际论文数量是否适合首页。

允许没有合格论文的日期，不使用旧论文补数量。

### 2.6 重大修订

只有以下变化可作为新的前沿信号：

- 新定理；
- 新方法；
- 新实验；
- 主结论实质变化。

一般扩写、语言润色、参考文献更新、元数据修正和普通勘误不算重大修订。

### 2.7 旧预印本的新期刊版本

旧预印本首次正式在线发表期刊版本时，可以重新进入首页，标记为 `journal-publication`。

页面同时显示：

- 期刊正式发表日期；
- 预印本首次公开日期。

### 2.8 核心与相邻方向比例

以统一质量门槛为前提，大致保持：

- 70--80% 核心结构前沿；
- 20--30% 可解释的相邻前沿。

这不是硬配额，不得为了比例降低标准。

### 2.9 自动内容的说明

不再给每篇论文重复显示“自动整理”标签。

首页和周归档顶部统一说明：

> 论文说明由自动流程整理，并经元数据核验；除特别标注外，不代表编辑已经完整阅读或逐句审校。

数据中继续保留 `review_status`，以便以后对人工审校条目单独标记。

### 2.10 原 W28、W29 阅读链

归档重写后，原阅读链只保留在 Git 历史：

- 不建立公开 legacy 页面；
- 不搬入其他公共栏目；
- 不将其包装成人工审核材料。

### 2.11 Core topics 暂停

本轮不新增或扩写 `Core topics / 核心主题`。

以后只有在编辑准备亲自审核某个具体小主题时才继续建设。

## 3. 关注主题

组织原则是“可积结构 + 创新”，不是固定方程名单。

### 3.1 核心结构前沿

- 谱问题、Lax pair、零曲率表示和 monodromy；
- direct/inverse scattering 和散射数据；
- 可积层级、recursion operator、Hamiltonian 与 Poisson 结构；
- Riemann--Hilbert problem、nonlinear steepest descent、soliton resolution 和临界渐近；
- tau function、Darboux/Bäcklund、Gram/Wronskian/Pfaffian 结构；
- finite-gap、谱曲线、Baker--Akhiezer 函数和退化极限；
- Painlevé、等单值变形和特殊函数结构；
- 离散可积系统、结构保持离散化、可积几何和 Yang--Baxter maps。

### 3.2 可解释的相邻前沿

- 量子可积性、Bethe Ansatz、quantum groups、Yangians 和 quantum spectral curves；
- generalized hydrodynamics、soliton gas、integrable turbulence 和 wave kinetics；
- 可积概率、随机矩阵、integrable kernels、loop equations 和精确分布；
- 实验、nonlinear Fourier methods、反问题、soliton tomography 和数据驱动发现可积结构。

相邻方向只有在可积结构直接参与主结果，并且能够向目标读者清楚说明桥梁时才收录。

## 4. 首页和归档的页面设计

### 4.1 总体原则

借鉴成熟论文列表的高信息密度，但不复制其数据库、搜索系统和完整功能。

默认状态下，一屏应能看到多篇论文。页面紧凑依靠“折叠详细内容”，而不是删减必要解释。

### 4.2 每篇论文默认显示

每个紧凑条目显示：

1. 实际研究事件日期；
2. 核心前沿或相邻前沿；
3. 一到两个官方 arXiv 分类，如 `nlin.SI`、`math.PR`、`math-ph`；
4. 最多两个受控的结构标签；
5. 作者；
6. 论文标题；
7. 约 100--180 个汉字的论文概览；
8. arXiv、PDF、DOI 或期刊页面链接；
9. “展开研究内容与创新”入口。

不显示：

- BibTeX 按钮；
- 逐篇“自动整理”标签；
- 自由生成的模糊标签；
- 旧式“推荐于某日”。

### 4.3 展开后的详细说明

展开区统一保留三个部分：

- **研究问题与主要结果**
- **可积结构与方法**
- **创新**

每部分通常写 2--4 句。展开后的总长度约 350--600 个汉字，使读者不打开 PDF 也能理解论文的问题、结构作用和新增内容。

“创新”只说明相对于已有工作的新增部分，不夸大论文影响，不预测长期意义。

### 4.4 时间组织

首页默认展示最近 30 天，并按周分组：

- 本周；
- 上周；
- 更早的周。

论文自身继续显示准确日期，但第一版不提供逐日日期下拉筛选。

第一版只保留：

- 全部；
- 核心前沿；
- 相邻前沿。

最近 7 天、14 天、30 天按钮暂不实现；先观察完整 30 天样本的数量和浏览体验。

### 4.5 页面层次

#### 首页

- 最近 30 天的紧凑论文流；
- 按周分组；
- 核心/相邻筛选；
- 默认折叠详细说明。

#### Radar 归档首页

只显示周目录，例如：

```text
2026-W29 · 2026-07-13 至 2026-07-19 · 收录 1 篇
2026-W28 · 2026-07-06 至 2026-07-12 · 收录 11 篇
```

不在归档首页重复展开全部论文。

#### 每周页面

- 使用与首页相同的论文条目组件；
- 只显示该周最终入选论文；
- 按实际事件日期排序；
- 页面底部附简短的每周筛选说明。

#### 其他页面

- `Core topics` 暂不改；
- `Group work` 暂不改；
- `About` 增加选稿原则和自动整理说明；
- `Resources` 只增加一条 Leonid Petrov 网站链接，不单独设标题，也不接入其 feed。

## 5. 标签规则

标签必须帮助读者识别来源分类或可积结构，不能由模型自由创造新术语。

### 5.1 官方分类标签

直接使用 arXiv 官方分类，不翻译、不改写，例如：

- `nlin.SI`
- `nlin.PS`
- `math.PR`
- `math-ph`
- `quant-ph`

默认显示 primary category，必要时再显示一个重要 cross-list 分类。

### 5.2 结构标签

结构标签来自固定词表。第一版候选词表包括：

- Lax pair
- inverse scattering
- Riemann--Hilbert problem
- nonlinear steepest descent
- Hamiltonian structure
- tau function
- Darboux transformation
- finite-gap
- spectral curve
- Painlevé
- isomonodromy
- Yang--Baxter equation
- Bethe Ansatz
- integrable discretization
- soliton gas
- generalized hydrodynamics
- integrable probability
- inverse problem
- loop equations
- Pfaffian structure

每篇默认最多显示两个结构标签。

不使用来源不清或含义不稳定的压缩词，例如临时创造的“谱控制”。

新增标签必须满足：

1. 是论文或领域中已经稳定使用的术语；
2. 能在论文摘要、正文或正式元数据中找到明确依据；
3. 与现有标签不重复；
4. 加入固定词表后再使用。

## 6. 数据维护方式

不使用 SQL 数据库，也不手工分别维护首页和周页面。

`data/papers.yml` 和 `data/editions.yml` 继续作为轻量结构化数据源。自动流程更新 YAML，人工通过 PR 审核，renderer 统一生成公开页面。

试运行阶段在 `data/editions.yml` 增加：

```yaml
- paper_id: "arxiv:xxxx.xxxxx"
  signal_date: "2026-07-19"
  signal_type: "new-preprint"
  frontier_class: "core"
  review_status: "automated"
  arxiv_categories:
    - "nlin.SI"
  structure_tags:
    - "Riemann--Hilbert problem"
    - "nonlinear steepest descent"
  summary: >-
    默认显示的 100--180 字论文概览。
  main_result: >-
    研究问题与主要结果。
  integrable_structure: >-
    可积结构与方法。
  innovation: >-
    创新。
```

候选值：

- `signal_type`: `new-preprint`, `major-revision`, `journal-publication`；
- `review_status`: 当前使用 `automated`，以后可增加 `human-reviewed`；
- `frontier_class`: `core`, `adjacent`。

页面全部由 renderer 生成，禁止手工修改生成后的论文列表。

验证脚本需要检查：

- `paper_id` 唯一；
- arXiv ID、DOI 和规范化标题去重；
- `signal_date` 和必填说明存在；
- arXiv 分类格式合法；
- 结构标签来自固定词表；
- 首页和周归档排序一致；
- renderer 重复运行结果一致。

完成回溯改写和若干期试运行后，再决定是否需要 `data/frontier.yml`。

## 7. 每日更新与每周筛选说明

### 7.1 每日流程

每日执行“检查”，不要求每日发布：

1. 检查 arXiv 相关分类；
2. 做少量跨分类结构查询；
3. 去重；
4. 阅读标题和摘要；
5. 对强候选核验论文正文、arXiv 版本和出版信息；
6. 按统一标准筛选；
7. 有合格论文才更新 PR；
8. 没有合格论文则不增加条目；
9. 不补旧背景论文。

### 7.2 每周补查

每周额外：

- 检查正式期刊发表和 DOI 信息；
- 汇总本周候选和最终收录数量；
- 记录主要来源类型和覆盖限制。

Petrov 网站只作为 `Resources` 中的成熟专题资源，必要时人工查漏；不自动同步其 feed，不依赖其数据。

### 7.3 公开的每周筛选说明

每周页面底部只公开聚合信息：

- 检查的主要来源类型；
- 去重候选数量；
- 完成标题筛选和摘要核验的数量；
- 最终收录数量；
- 核心与相邻方向数量；
- 本周覆盖限制。

不公开逐篇未收录名单，不对未收录论文作负面评价。

## 8. 旧论文处理

从首页移除、从公开归档移除、从 `data/papers.yml` 删除是不同操作。

- 可靠旧论文元数据可以保留用于去重；
- 未人工审核的旧背景论文不自动进入 Core topics；
- 为凑数量、重复或关系牵强的推荐从公开 edition 删除；
- 重复、错误或无法核实的记录在确认无引用后才从 registry 删除；
- 年代久远本身不是删除 bibliographic record 的理由。

首页和周归档共用同一个论文条目 renderer 和同一套 CSS。

第一版预计修改：

- `data/editions.yml`

- `docs/stylesheets/radar.css`

- renderer 生成的首页和 radar 页面

不修改导航，不新增栏目，不扩写 Core topics。
