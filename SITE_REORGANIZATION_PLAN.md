相关筛选结果见 [`W28_W29_RETROSPECTIVE_SCREENING_REPORT.md`](W28_W29_RETROSPECTIVE_SCREENING_REPORT.md)。

## 1. 项目边界

本站要做的是：

> 面向熟悉可积 PDE 和谱方法的研究者，提供一个轻量、精选、可解释的近期论文雷达。

核心价值是帮助读者减少需要亲自筛选的论文数量，并清楚说明：

1. 论文研究了什么问题、得到什么主要结果；
2. 可积结构在论文中起什么作用；
3. 论文的创新是什么。

本站不建设 arXiv 镜像、完整论文数据库、全历史语义检索、作者优先系统或自动旧文阅读链。今后的功能只有在直接帮助筛选近期论文或解释论文结构与创新时才考虑。

## 2. 入选标准

公开论文原则上同时满足：

1. 可积性是核心结构或核心方法，而不是附带背景；
2. 有清楚、可核验的创新；
3. 能从 arXiv、论文正文或出版商页面准确说明；
4. 能向目标读者解释结构如何产生结果；
5. 提供超过直接浏览标题和摘要的编辑价值。

### 2.1 两种入选路径

不再把“必须提出新的可积结构”作为唯一入选方式。允许两种同等有效、但作用不同的贡献类型：

#### 结构推进

论文提出、揭示或实质扩展可积结构或方法，例如：

- 新的或实质扩展的 Lax pair、谱问题、层级、Hamiltonian/Poisson 结构；
- 新 RHP、tau function、谱曲线、Painlevé 或 Yang--Baxter 结构；
- 保持核心结构的可积离散化；
- 由新结构直接得到新定理、分类、渐近或重构机制。

#### 结构驱动创新

论文使用已有或已知的可积结构，但该结构对新的主要结果不可替代，例如：

- 由 Darboux、IST、RHP、散射数据或精确解结构得到系统的新分类；
- 构造任意阶或任意形态的解族，而非少量低阶例子；
- 揭示新的渐近、拓扑、稳定性或物理机制；
- 产生新的反演、控制、实验或统计结果。

这两个名称只是贡献方式，不是质量等级。论文不能因为某一周数量较少而降低门槛；但也不能因为没有提出全新的 Lax pair 就自动排除。

### 2.2 通常不收录

- 标准变换的机械套用；
- 只有少量低阶显式解和参数图；
- 没有结构性结论的数值展示；
- 熟悉方法几乎无概念变化地迁移；
- 可积性只在引言或背景出现；
- 无法可靠说明研究问题、结构作用和创新。

不公开使用“较弱”“不值得读”“创新性差”等评价。未收录只表示不符合本站本轮范围或当前说明门槛。

## 3. 时间与范围

### 3.1 论文事件日期

公开排序依据：

- 首次公开日期；
- 重大修订日期；
- 首次正式在线期刊发表日期。

本站收录日期只属于编辑历史，不参与首页排序。

### 3.2 首页窗口

先完成 2026-06-20 至 2026-07-19 的完整样本，再判断最近 30 天是否合适。允许没有合格论文的日期，不使用旧论文补数量。

### 3.3 重大修订

只有新定理、新方法、新实验或主结论实质变化才作为新的前沿信号。语言润色、参考文献更新、元数据修正和普通勘误不算重大修订。

### 3.4 期刊版本

旧预印本首次正式在线发表期刊版本时，可以重新进入首页。页面同时显示期刊发表日期和预印本首次公开日期。

## 4. 页面设计

### 4.1 总体形式

借鉴成熟论文列表的高信息密度，但不复制其数据库和搜索系统。默认状态下一屏能看到多篇论文；页面紧凑依靠折叠详细内容，而不是删减必要解释。

### 4.2 每篇论文默认显示

1. 实际研究事件日期；
2. 贡献类型：`结构推进` 或 `结构驱动创新`；
3. 一到两个官方 arXiv 分类，如 `nlin.SI`、`nlin.PS`、`math.PR`；
4. 最多两个受控结构标签；
5. 作者和标题；
6. 约 100--180 个汉字的论文概览；
7. arXiv、PDF、DOI 或期刊页面链接；
8. “展开研究内容与创新”入口。

不显示 BibTeX、逐篇“自动整理”标签、自由生成的模糊标签或“推荐于某日”。

### 4.3 展开说明

统一使用：

- **研究问题与主要结果**
- **可积结构与方法**
- **创新**

每部分通常写 2--4 句，总长度约 350--600 个汉字。“创新”只说明相对于已有工作的新增部分，不夸大长期影响。

### 4.4 首页与归档

首页：

- 最近 30 天；
- 按周分组；
- 每篇显示准确日期；
- 第一版不做逐日日期筛选；
- 可按贡献类型筛选；
- 所有详细说明默认折叠。

Radar 归档首页只显示周目录，不重复展开全部论文。

每周页面：

- 复用首页的论文条目组件；
- 只显示该周最终入选论文；
- 按实际事件日期排序；
- 页面底部附简短、聚合的筛选说明。

其他页面：

- `Core topics` 和 `Group work` 暂不改；
- `About` 增加选稿原则和自动整理说明；
- `Resources` 只增加一条 Leonid Petrov 网站链接，不接入其 feed。

## 5. 标签规则

### 5.1 arXiv 分类

直接使用官方分类，不翻译、不改写，例如：

- `nlin.SI`
- `nlin.PS`
- `math.PR`
- `math-ph`
- `quant-ph`

默认显示 primary category，必要时再显示一个重要 cross-list 分类。

### 5.2 结构标签

结构标签来自固定词表，例如：

- Lax pair
- inverse scattering
- Riemann--Hilbert problem
- nonlinear steepest descent
- Hamiltonian structure
- tau function
- Darboux transformation
- finite-gap
- spectral curve
- Painlevé
- isomonodromy
- Yang--Baxter equation
- Bethe Ansatz
- integrable discretization
- soliton gas
- generalized hydrodynamics
- integrable probability
- inverse problem
- loop equations
- Pfaffian structure
- topological vector potential

每篇默认最多显示两个结构标签。禁止模型临时创造“谱控制”之类来源不清、含义不稳定的术语。新增标签必须能在论文或领域正式用语中找到依据，并先加入词表。

## 6. 数据维护

不使用 SQL 数据库，也不手工分别维护首页和周页面。`data/papers.yml` 和 `data/editions.yml` 继续作为轻量结构化数据源，自动流程更新 YAML，人工通过 PR 审核，renderer 统一生成公开页面。

试运行字段：

```yaml
- paper_id: "arxiv:xxxx.xxxxx"
  signal_date: "2026-07-19"
  signal_type: "new-preprint"
  contribution_type: "structure-advance"
  review_status: "automated"
  arxiv_categories:
    - "nlin.SI"
  structure_tags:
    - "Darboux transformation"
  summary: >-
    默认显示的论文概览。
  main_result: >-
    研究问题与主要结果。
  integrable_structure: >-
    可积结构与方法。
  innovation: >-
    创新。
```

候选值：

- `signal_type`: `new-preprint`, `major-revision`, `journal-publication`；
- `contribution_type`: `structure-advance`, `structure-enabled-innovation`；
- `review_status`: `automated`，以后可增加 `human-reviewed`。

验证脚本检查：

- `paper_id`、arXiv ID、DOI 和规范化标题去重；
- 日期和必填说明存在；
- arXiv 分类格式合法；
- 结构标签来自固定词表；
- 首页和周页面排序一致；
- renderer 重复运行结果一致。

## 7. 自动内容说明

不再给每篇论文重复显示“自动整理”。首页和周页面顶部统一说明：

> 论文说明由自动流程整理，并经元数据核验；除特别标注外，不代表编辑已经完整阅读或逐句审校。

数据中继续保留 `review_status`。

## 8. 日常流程

每日执行检查，不要求每日发布：

1. 检查 arXiv 相关分类；
2. 做少量跨分类结构查询；
3. 去重并阅读标题和摘要；
4. 对强候选核验论文正文、版本和出版信息；
5. 按统一标准筛选；
6. 有合格论文才更新 PR；
7. 没有合格论文则不增加条目；
8. 不补旧背景论文。

每周额外检查期刊发表和 DOI，汇总候选与收录数量，记录来源类型和覆盖限制。公开周报只展示聚合信息，不列未收录论文，也不作负面评价。
