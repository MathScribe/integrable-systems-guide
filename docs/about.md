# About / 关于

This project is a small research-oriented guide for integrable systems and related methods.

这是一个研究向资料入口，主要整理可积系统、反散射、黎曼--希尔伯特方法、非线性波和相关课题组公开资料。

## Site model / 页面职责

本站的主要交付物是持续维护的论文阅读队列，而不是一份以当天日期为中心的新闻列表。

- 首页显示最近三期中优先级较高的论文，不按日期分块，也不复制完整简报；
- 归档首页按周组织，提供主题、篇数和周级入口；
- 周归档保留每日 edition，是推荐注释的永久正文；
- `data/papers.yml` 保存当前书目信息，`data/editions.yml` 保存推荐日期、阅读定位和编辑注释；
- `docs/radar/latest.md` 仅作为旧链接的兼容入口，不再保存一份重复正文。

每日运行是维护频率，不是网站的信息架构。日期表示本站推荐时间，不表示论文在当天提交或发表。

## Curation policy / 策展原则

每个条目应说明论文实际做了什么，以及为什么值得现在阅读。理由尚不清楚的内容保留在候选阶段，不直接公开推荐。

每期可以混合近期论文、近期漏读、正式期刊版本、课题组相关工作、核心旧文、方法基础、综述地图和相邻方向。论文之间确有依赖关系时可以说明阅读顺序；没有严格递进关系时，不强行包装成一条故事线。

旧文优先链接正式期刊或 DOI 记录。已有论文的 arXiv 升版只更新当前元数据，不自动成为新的推荐。

## Annotation format / 注释格式

周归档中的论文条目使用稳定格式：

1. 未链接的论文题目；
2. 完整作者列表，以及 arXiv、DOI 或期刊标识上的外部链接；
3. 固定的阅读定位与主题标签；
4. “做了什么”：摘要依据的模型、问题、方法和结果；
5. “为什么值得读”：它与当前研究方向、方法链条或既有文献的关系。

注释应独立成立，不依赖“本期第一篇”“最后一篇”或其他容易因排序变化而失效的表述。

## Metadata and history / 元数据与推荐历史

`data/papers.yml` 是当前书目事实的注册表，用 arXiv ID、DOI 和规范化题名去重。作者、题名、版本、修订日期和期刊信息可以随来源记录更新。

`data/editions.yml` 记录本站的推荐历史。每次推荐引用稳定的 `paper_id`，并保存阅读定位、首页优先级和当次注释。页面由这两个数据文件生成，CI 检查数据引用、去重、周编号和生成结果是否一致。

迁移期间，`papers.yml` 中旧有的 `featured_on` 字段暂时保留为兼容镜像；页面与归档以 `editions.yml` 为准，后续可在历史数据稳定后删除镜像字段。

## AI-assisted curation / AI 辅助整理

AI 用于检索、去重、元数据清洗、初步分类和文字整理。公开推荐主要依据题名、摘要、版本记录、期刊信息和公开元数据筛选，不代表已经完成全文阅读或数学正确性核验；数学结论请以原论文为准。
