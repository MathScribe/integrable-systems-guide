## Candidate queue

Candidate resources should be kept separate from the core table until they have been checked. A candidate entry should record the source, topic placement, reason for possible inclusion, and review status.

Typical candidate sources include researcher homepages, course pages, lecture notes, summer school pages, workshop pages, book companion pages, software repositories, and Zotero-derived reading lists.

## Maintenance rule

Promote a resource into the core table only after checking the link, mathematical relevance, expected background, and current usefulness. If a resource is merely famous but not useful for a reader's next step, leave it out or keep it in the candidate queue.
