# Paper data and selection / 论文数据与筛选方法

本页说明研究雷达的数据来源、日期、去重、筛选和论文注释规则。

## 公开内容的职责

研究雷达累计保存最终收录论文。首页默认显示最新一周，并提供周翻页、全文搜索和按月份组织的“查看全部”模式。它不是 arXiv 镜像，也不要求每天产生内容。

公开页面只收录三类研究事件：

- `new-preprint`：新预印本首次公开；
- `major-revision`：新增主要定理、核心方法、实验、实质分析或改变主要结论的重大修订；
- `journal-publication`：首次正式在线期刊发表。

普通文字润色、参考文献、排版、作者顺序、元数据和小范围勘误不构成重大修订。

## 日期含义

- `submitted`：arXiv 记录的预印本首次提交日期；
- `updated`：当前预印本版本日期；
- `published`：首次正式在线期刊发表日期；
- `signal_date`：当前公开雷达用于排序和周分组的公开事件日期；新预印本采用 arXiv 官方列表的 announcement date；
- `metadata_checked_at`：本站最近一次核验书目信息的日期。

本站何时发现论文不作为公开排序依据。

日期沿用来源的官方记录，不统一换算成本地日历日。arXiv 的 `submitted` 和 `updated` 保留版本页记录的 UTC 日期；新预印本用于排序和周分组的 `signal_date` 采用 arXiv 官方列表的 announcement date。期刊事件采用出版商标注的首次在线发表日期。每日检索按北京时间运行，执行时间只表示本站何时检查，不改变论文的公开事件日期。

[arXiv 官方公告规则](https://info.arxiv.org/help/availability.html)说明，arXiv 通常只在周日至周四发布公告，周五和周六没有常规公告。例如，在美国东部时间周四 14:00 至周五 14:00 之间提交的论文，会在周日 20:00 公开并进入周一的新论文列表。因此卡片可能同时显示“周一公开”和“上周五 UTC 提交”；两者分别表示公开批次和作者完成提交的时间，并不冲突。

## 一篇论文只保留一个公开条目

`data/papers.yml` 中同一工作只保存一份当前书目记录，不为 arXiv 和 DOI 创建两条重复记录。

`data/editions.yml` 的 `frontier` 中同一论文只保存一个公开条目。已经收录的预印本发生重大修订或首次正式发表时，更新原条目：

- 保留最初 arXiv ID 和提交日期；
- 补充或更新 DOI、期刊、卷期、页码或文章号；
- 将 `signal_type` 和 `signal_date` 更新为新的合格事件；
- 只在新事件改变论文结果时调整说明。

早期状态由 Git 历史保留，公开页面不显示重复卡片。

## 筛选原则

主题相关性是宽松入口，创新强度是决定因素。论文可以推进新的可积结构或方法，也可以利用已有可积结构得到清楚、系统且非平凡的新结果。

引力、几何、概率、随机矩阵、量子多体、统计物理、光学和实验等方向，只要可积性对核心结论不可替代，均可进入候选范围。

公开文字只说明入选论文的研究问题、可积结构和创新，不发布未入选论文名单，也不使用贬值性评价。

## 数据怎样产生

候选发现和最终核验是两个不同阶段。发现阶段尽量避免遗漏，最终页面只显示经过筛选和核验的论文。

### 每日预印本发现

1. 完整检查 arXiv `nlin.SI` 和 `nlin.PS` 的新增及修订记录；
2. 在数学物理、应用数学、概率、几何、组合、量子多体、统计物理、引力和光学等分类中做有限的跨分类检索；
3. 检索同时使用结构词和结果词。前者包括 Lax pair、inverse scattering、Riemann--Hilbert、Painlevé、Yang--Baxter、Bethe ansatz、Hamiltonian 和 integrable discretization；后者包括 classification、asymptotics、transition、inverse problem、control、experiment、transport 和 topology；
4. 普通网页搜索和专题资源只用于发现遗漏线索，不直接决定收录，也不作为最终元数据来源。

### 每周期刊发表检查

1. 使用 Crossref 按日期、完整题名、作者和结构关键词形成候选，并检查现有预印本是否获得 DOI；
2. 查看相关期刊的 Latest articles 或 Online first 页面，补充 Crossref 尚未稳定收录的论文；
3. 打开 DOI 对应的出版商页面，核对期刊名、作者、卷期、页码或文章号以及首次在线发表日期；
4. 区分首次在线发表与后来分配卷期。仅分配卷期不构成新的研究事件。

Crossref 和网页搜索负责发现或匹配；最终日期和期刊信息以出版商记录为准。论文结论则回到 arXiv 摘要、论文 PDF 或正式全文核验。

### 去重、筛选和说明

候选按 arXiv ID、DOI 和规范化题名去重。主题相关性是宽松入口，最终判断集中在两个问题：可积结构是否对主要结果起实质作用，以及论文是否给出可以清楚说明的新结构、新方法、新分类、新渐近、新实验或其他非平凡结果。

每周大约二十篇只能作为检查发现覆盖是否异常的参考，不能作为收录配额。某周数量明显偏少时会先复查分类、查询词和期刊覆盖；复查后仍没有足够论文时，页面保留真实结果，不用旧文或一般性工作补足数量。

## 注释格式

每篇论文包含：

- `summary`：两至三句论文概览，概括研究对象、主要结果和最值得注意的部分；
- `main_result`：研究问题、结果层次及其解析、数值或实验依据；
- `integrable_structure`：可积结构与方法如何实际参与结果；
- `innovation`：相对于已有工作的具体新增内容。

四个字段不得重复完整句子。`summary` 是结果层面的压缩，`main_result` 负责展开结果内部结构，`integrable_structure` 解释机制，`innovation` 给出有来源支持的比较；不能从同一句摘要机械生成四种改写。

结构标签必须从受控词表选择。官方 arXiv 分类和本站结构标签承担不同作用，不应混用。

## AI 与人工审核

AI 用于候选发现、元数据核验、初步筛选、文字整理、页面生成和一致性检查。除非特别注明，公开说明不表示编辑已经完成全文逐句审校或独立验证数学证明。

每次实际变更通过可审核 PR 提交。CI 成功不代表合并批准；是否合并由项目维护者明确决定。
